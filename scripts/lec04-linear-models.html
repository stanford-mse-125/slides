Lecture 4: Linear Models — Facilitation Script

Duration: 80 minutes Primary dataset: NYC Airbnb (5-listing toy example + 28,761 full dataset) Structure: Opening (5 min) → Block 1: Vectors (15 min + 5 min) → Block 2: Linear models + regression (15 min + 5 min) → Block 3: Multiple regression + R² (15 min + 5 min) → Closing (5 min) → Buffer (5 min)

Opening (2 min)

Hook (slide 2, dark): “How much is a bathroom worth?” — an Airbnb host adding a second bathroom. We’ll answer this by the end of the lecture.

Agenda (slide 3): vectors, linear combinations, column space, regression, R². Jump straight into Block 1.

Block 1: Vectors in Data (slides 9-16, ~20 min)

Key narrative

Data points are vectors. Once we see them as vectors, we can measure distance (norm), similarity (inner product), and direction (cosine similarity). This language will let us understand what a linear model does.

Slide-by-slide

Slide 9 (two views): Show the toy table. Two views: row = point, column = vector. Emphasize this is the same data seen two ways.

Slide 10 (scatter): Point at “Listings 1 & 5” — same features, same point. Listing 3 is farthest away.

Slide 11 (difference vector): Code on slide — no context switch needed. The difference vector says how they differ.

Slide 12 (norm): Definition box. “Generalizes the Pythagorean theorem.” Show the code: np.linalg.norm(diff) = 2.24.

Slide 13 (inner product): Definition box. Positive = same direction, negative = opposite, zero = perpendicular. Show code.

Slide 14 (cosine similarity): Formula, then code. “0.98 means very similar proportions.”

Discussion 1 (slide 15): Poll — which features carry independent info?

  • Format: Poll + predict-then-reveal (3 min)
  • Setup: “beds vs bedrooms — high or low cosine? beds vs reviews — high or low?”
  • If stuck: “If you know bedrooms, can you guess beds? What about reviews?”
  • Key insight: beds/bedrooms = 0.65 (redundant), beds/reviews = 0.10 (independent). Independent features are more valuable.
  • Reveal: Show slide 16 with the answer table.

Block 2: Linear Models + Regression (slides 17-30, ~20 min)

Key narrative

A linear model is a weighted sum of feature columns. The set of all possible predictions = column space. Regression finds the closest point in that space to the true prices.

Slide 17 (dark): “what can your model reach?”

Slide 18 (linear combination code): Show the computation — 100×bedrooms + 50×bathrooms. Code + output on the slide. “This is ALL a linear model does.”

Slide 19 (definition): Linear combination definition.

Slide 20 (span image): The lattice of all integer-weight combinations. “With real weights, you can reach every point in the plane.”

Discussion 2 (slide 21): Think-pair-share — what if vectors are parallel?

  • Format: Think-pair-share (3 min)
  • Setup: “Can you reach every point in R²? What if v₁ and v₂ point the same direction?”
  • If stuck: “What shape do you get from multiples of a single vector?”
  • Key insight: Non-parallel = span all of R². Parallel = only a line. Redundant features don’t help.

Slide 22 (3D projection): Crucial image. “Two vectors in R³ span a plane. The target is off the plane — the closest point is the projection.” Point out the right-angle mark.

Slides 23-24 (column space): Definition, then code. X @ beta computes everything at once.

Slide 25 (dark): “regression = projection”

Slides 26-28 (real data): Load 28,761 listings. Show scatter + regression line. ŷ = 71 + 49 × bedrooms.

Slide 29 (inline Q/A): “What does the intercept mean?” Reveal: $71 for a studio.

Slide 30 (why squared?): Brief — two reasons (projection geometry, conditional mean). “We could choose differently — and we will, in Ch 13.”

Block 3: Multiple Regression, R², and Diagnostics (slides 31-43, ~20 min)

Key narrative

Adding features expands the column space. Multiple regression lets us “hold constant.” The normal equations explain why. R² measures fit quality. Residual plots reveal what the model misses.

Slides 31-32 (regression to the mean): Galton story. “Since |r| < 1, predictions are pulled toward the mean.” Connect to the name “regression.”

Discussion 3 (slide 33): Predict-then-reveal — basketball player

  • Format: Predict-then-reveal (3 min)
  • Setup: “Player scores 45 (average: 25). Predict next game?”
  • If stuck: “Is the 45 more likely a fluke or the new normal?”
  • Key insight: Below 45, above 25. Coaches who overreact to extreme games are ignoring regression to the mean.

Slides 34-35 (multiple regression): Code + coefficient table. “Each bathroom ~$22, private room −$90.”

Slide 36 (answer!): “~$22/night for that second bathroom.” R² jumped from 0.16 to 0.45.

Discussion 4 (slide 37): Poll — sign of reviews coefficient?

  • Format: Poll (2 min)
  • Setup: “Positive, negative, or zero?”
  • Key insight: Negative. Cheaper listings get booked more → more reviews. Association ≠ causation.

Slides 38-39 (orthogonality): Column space cartoon + normal equations. “The residual has zero dot product with every feature.”

Slide 40 (verify in code): Show the orthogonality check — both dot products ≈ 0.

Slide 41 (R² formula): Pythagorean decomposition. Total = explained + residual.

Discussion 5 (slide 43): Think-pair-share — can R² decrease?

  • Format: Think-pair-share (4 min)
  • Setup: “Think about the geometry: adding a column makes the column space…”
  • If stuck: “Can a projection onto a bigger space be farther from the target?”
  • Key insight: Training R² always increases. Test R² can decrease (overfitting, Ch 7).

Slide 42 (residual plots): Fan shape = heteroscedasticity. “The model misses something.”

Closing (slides 44-47, ~5 min)

Slide 44 (key takeaways): Walk through all six. The chain: vectors → linear combinations → column space → projection → orthogonality → R².

Slide 45 (what we can’t answer yet): Three open questions → forward pointers to Ch 5, 7, 12.

Slide 46 (logistics): Read Ch 4, HW 1 due Friday, Quiz 2 next Wednesday.

Vignette notes

Galton’s regression to the mean

Francis Galton studied 928 father-son pairs in the 1880s. Sons of tall fathers were tall but less extreme; sons of short fathers were short but less extreme. He called it “regression toward mediocrity” and the method he used to quantify it became “regression.” (Salsburg, The Lady Tasting Tea, Ch. 4.)

“All models are wrong, but some are useful” — George Box

The projection ŷ won’t equal y exactly. The model is wrong. But it’s the best approximation within the column space. Use this framing when introducing the column space.

From the Cornell version: scaffold computational complexity with inline polls. “How many flops to compute u·v?” → “How many for Xw?” Each builds on the previous. Good for math-heavy sections if you need to fill time.
